Explaning 802.11 Beacon frames

In 802.11 wireless networks, there are 4 types of frames, namely, Management, Control, Data, and Extension frames. In this post, we will discuss about 802.11 Beacon frames, which is of management type.
80 00
Frame Control (0x0008)
In the beacon frame,the first 2 bytes are frame control field. Version, Type and Subtype are always present in Frame Control Packet.
  • .... ..00= Version: 0
  • .... 00..= Type: Management frame (00)
  • 1000 ....= Subtype: 8
  • Version:- It is 2 bit field. Always 00 for WLAN Frames WLAN (PV0) and set to 01 for 802.11ah (PV1). 802.11ah is also known as Wi-Fi HaLow â€“ sub 1 GHz long range, low power wireless technology standardized by Wi-Fi Alliance in 2017 for IoT devices. It uses 900 MHz license-exempt bands to provide extended-range Wi-Fi networks, compared to conventional Wi-Fi networks operating in the 2.4 GHz, 5 GHz and 6 GHz bands.
  • Type:- It is also 2 bit field. Type decides the type of Packet. Packet can be management, Control,Data or Extended. 00 defines management frame, 01 defines control frame, 10 defines data frame and 11 defines extended frame
  • Subtype:- It is also 2 bit field. This field defines the subtype of management, control, data and extended frame like Association request, Association response, probe request, probe response, beacon, authentication, deauthentication are subtype of management type. RTS/CTS, ACK, Block-ACK,PS-Poll etc are subtype of control frames. QOS, CF-ACK, CF-Poll, QOS Null etc are subtype of Data type frame. Extended type is new to 802.11. DMG beacon and S1G beacon are subtype of Extension frame.
Flags: 0x00
  • .... ..00= DS status: Not leaving DS or network is operating in AD-HOC mode (To DS: 0 From DS: 0) (0x0)
  • .... .0..= More Fragments: This is the last fragment
  • .... 0...= Retry: Frame is not being retransmitted
  • ...0 ....= PWR MGT: STA will stay up
  • ..0. ....= More Data: No data buffered
  • .0.. ....= Protected flag: Data is not protected
  • 0... ....= +HTC/Order flag: Not strictly ordered
  • DS Status:- It is 1 bit field. To and from DS is 0 means communication is happening in AD-HOC mode. Both To DS and From DS are 1 bit each.
  • More Fragments:- It is 1 bit field.
  • Retry:- It is 1 bit field. This bit is ON if due to interference frames got corrupted and had to be retransmitted. Frame is not being transmitted means there was no collision.
  • PWR MGT:- It is 1 bit field. It was introduced to save the power of end devices. When this bit is ON, it means end device is sleeping and access point buffer any data related to it.
  • More Data:- It is also 1 bit field. More data bit is used only in management and data frames. This bit is set to 0 in control frames.
  • Protected Flag:- It is 1 bit field. Protected frame bit is only for control frames. Control frames may not be encrypted, so this frame bit is always 0.
  • +HTC/Order flag:- It is 1 bit field. control frames can't be out of order. Therefore, this bit is set to 0.
